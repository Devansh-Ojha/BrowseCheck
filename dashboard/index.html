How it works

        
Three steps. Zero compromises.

        
The entire security layer is transparent to the user and adds less than 500ms to each agent action.

      

      

        

          
1

          
Agent visits the page

          
The AI reads everything on the page, including visible content, hidden HTML comments, invisible divs, and encoded payloads the user never sees.

          
→ navigate ai-hackberkeley.org

        

        

          
2

          
BrowseCheck intercepts

          
Before a single browser action fires, the hook pipeline runs in parallel: prompt injection scan, intent drift check, credential phish detector, cross-site carry-over analysis.

          
⚡ 4 hooks · ~340ms

        

        

          
3

          
Block or allow

          
Fail-closed: if any hook blocks, the action never reaches the browser. Legitimate actions from the visible page go through without friction.

          
✗ BLOCKED : injection

        

      

    

  


  
  

    

      

        
Under the hood

        
Purpose-built for the agentic web.

        
This isn't a wrapper around an existing tool. Every layer was designed specifically for the problem of agentic browser security.

      

      

        

          
LLM Core

          
Anthropic Claude

          
Every hook that needs to reason about intent runs a targeted Claude call with a purpose-built system prompt. Not a general chat; a specialized classifier.

        

        

          
Live Browser

          
Browserbase

          
Real Chromium browser execution in the cloud with live-view iframe streaming. The attack demo you're watching is a real browser, not a simulation.

        

        

          
Hook Pipeline

          
Parallel · Fail-Closed

          
All hooks run concurrently via asyncio.gather. If any hook blocks, the action is killed. A crashing hook emits an alert and defaults to allow, so there are no silent failures.

        

        

          
Real-Time Events

          
FastAPI + SSE

          
Every hook result, agent action, and site visit emits a SecurityEvent over a Server-Sent Events bus. The dashboard updates live with zero polling.

        

        

          
Novel Hook

          
Cross-Site Carry-Over

          
Our novel contribution: a stateful hook that tracks injections across sites and blocks actions on site B that were instructed by site A, an attack class no existing tool catches.

        

        

          
Architecture

          
Prevention, Not Detection

          
Every other security tool observes after the fact. BrowseCheck intercepts before the browser ever acts. The injected instruction never executes. Not even once.
